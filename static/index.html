Algorithms
How each algorithm works under the hood
sliding_windowZSET · O(log N)

Stores each request as a member of a Redis sorted set scored by timestamp-ms. Atomically prunes expired entries, counts remaining, then inserts if below limit.

ZREMRANGEBYSCORE key -∞ (now - window)
ZCARD key → count
if count < limit: ZADD key now member
-- all atomic via Lua script
no boundary burstO(log N)memory ∝ volume
fixed_windowSTRING · O(1)

Counts in fixed time buckets. Key is rl:fw:{client}:{floor(now/window)}. INCR + auto-expire. Fastest possible — one round-trip.

bucket = floor(now_ms / window_ms)
INCR rl:fw:{client}:{bucket}
if count == 1: EXPIRE key window+1
if count <= limit: allow
lowest latencyO(1) memory2× burst at boundary
token_bucketHASH · O(1)

Stores {tokens, last_refill} in a Redis Hash. Virtual refill on every read — no background process needed.

elapsed = (now - last_refill) / 1000
tokens = min(capacity, tokens + elapsed × rate)
if tokens ≥ 1: tokens -= 1; allow
HMSET key tokens {tokens} last_refill {now}
burst toleranceO(1) memory per clientsmooth refill
